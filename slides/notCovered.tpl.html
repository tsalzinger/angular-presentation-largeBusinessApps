Build system

What we use

  • TeamCity
  • gradle
  • node.js
  • gulp.js
  • bower
  • karma
- TeamCity: our CI - gradle: build automation evolved (programmatic as opposed to mvn), configurations and tasks - node.js: platform for all our js resource handling - gulp.js: a streaming build system (gulp vs. grunt == gradle vs. mvn) - bower: js dependency management tool (looking for alternatives) - karma: js test runner with support for jasmine, etc. and all browsers
